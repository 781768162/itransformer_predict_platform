# 新能源发电量与用电量预测平台

## 项目简介

本小组项目旨在利用 iTransformer 时序预测模型，构建一个新能源发电量与用电量的预测平台。项目后端采用微服务架构，以实现高内聚、低耦合的模块化设计。

### 技术架构

项目后端主要包含以下几个核心模块：

1.  **API 网关 (Golang)**
    *   使用 Gin 框架，作为系统的统一入口。
    *   负责处理用户认证，包括登录、注册及后续的请求鉴权。
    *   整合、处理来自不同服务的数据。
    *   作为代理与其他后端服务进行交互。

2.  **算法服务模块 (Python)**
    *   包含两个独立的 Python 算法服务。
    *   每个服务都基于 FastAPI 框架构建，提供轻量级的 RESTful API 接口。
    *   负责执行核心的 iTransformer 时序预测算法。
    *   与 API 网关进行数据交互，接收预测请求并返回结果。

3.  **部署 (Docker)**
    *   以上所有服务（API 网关、两个算法服务模块）都将通过 Docker 进行容器化部署，以简化环境配置、提高可移植性并实现快速伸缩。
